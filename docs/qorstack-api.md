# Qorstack API Spike

Ticket: 014

Checked against local Qorstack at `http://localhost:18080` on 2026-06-16. The service did not expose docs at `/docs`, `/swagger`, or `/api-docs`; the live OpenAPI document was available at `/openapi/v1.json`.

Authentication for API clients uses:

```http
X-API-Key: <QORSTACK_API_KEY>
```

## Endpoints Found

### `GET /openapi/v1.json`

Returns the OpenAPI JSON document.

### `GET /health`

Returns service health JSON.

### `POST /auth/login`

Request:

```json
{
  "email": "admin@example.com",
  "password": "admin"
}
```

Returns an auth response for the web app login flow. API client calls should use `X-API-Key` instead.

### `GET /templates`

Lists templates.

Query parameters include `status`, `search`, `projectId`, `pageNumber`, and `pageSize`.

Response shape:

```json
{
  "items": [
    {
      "templateKey": "bill",
      "name": "Bill",
      "activeVersion": {
        "version": 1,
        "filePath": ".../v1.docx",
        "previewFilePath": ".../v1_preview.pdf"
      }
    }
  ],
  "pageNumber": 1,
  "totalPages": 1,
  "totalCount": 1,
  "hasPreviousPage": false,
  "hasNextPage": false
}
```

### `POST /templates`

Uploads a new DOCX or XLSX template.

Query parameters:

- `name`: template display name
- `templateKey`: optional stable key
- `project_id`: optional project UUID
- `isAutoGeneratedVariable`: required boolean

Request body: `multipart/form-data` with `file`.

For DOCX upload, curl needed an explicit content type:

```sh
curl -H "X-API-Key: $QORSTACK_API_KEY" \
  -F "file=@templates/bill.docx;type=application/vnd.openxmlformats-officedocument.wordprocessingml.document" \
  "$QORSTACK_API_URL/templates?name=Bill&templateKey=bill&isAutoGeneratedVariable=false"
```

Response shape:

```json
{
  "templateKey": "bill",
  "name": "Bill",
  "activeVersion": {
    "version": 1,
    "filePath": ".../v1.docx",
    "previewFilePath": ".../v1_preview.pdf"
  }
}
```

### `GET /templates/{templateKey}`

Returns one template and its versions.

### `PUT /templates/{templateKey}`

Updates a template and can upload a replacement DOCX/XLSX file.

### `DELETE /templates/{templateKey}`

Deletes a template.

### `GET /templates/{templateKey}/download`

Returns a template download URL.

Response shape:

```json
{
  "url": "http://localhost:9100/template/..."
}
```

### `GET /templates/template-generate-key`

Returns a generated unique template key string.

### `POST /render/word/template`

Renders a stored Word template to PDF or DOCX and returns a presigned download URL.

Request shape:

```json
{
  "templateKey": "bill",
  "fileName": "bill.pdf",
  "fileType": "pdf",
  "replace": {
    "tenantName": "Example"
  },
  "table": null,
  "image": null,
  "qrcode": null,
  "barcode": null,
  "pdfPassword": null,
  "watermark": null,
  "zipOutput": false
}
```

Response shape:

```json
{
  "jobId": "2f392c76-53cd-480f-a9c4-40f39b9d2141",
  "downloadUrl": "http://localhost:9100/report/temp-download/...pdf?...",
  "expiresIn": 3600,
  "fileType": "pdf",
  "status": "success",
  "isZipped": false
}
```

### `POST /render/word/template-sandbox/{templateKey}`

Renders a stored Word template using its configured sandbox payload.

Optional query parameter: `fileName`.

Request body may be `null` or a JSON object.

Response shape matches `POST /render/word/template`.

### `POST /render/excel/template`

Renders a stored XLSX template and can output XLSX or PDF.

Request shape is similar to Word rendering:

```json
{
  "templateKey": "sheet",
  "fileName": "sheet.pdf",
  "replace": {
    "tenantName": "Example"
  },
  "table": null,
  "image": null,
  "qrcode": null,
  "barcode": null,
  "zipOutput": false
}
```

### `GET /render/jobs`

Lists render jobs.

Query parameters include `templateKey`, `status`, `fromDate`, `toDate`, `pageNumber`, and `pageSize`.

### `GET /render/jobs/{id}`

Returns one render job.

## Capability Matrix

| Capability | Supported by public Qorstack API? | Evidence |
| --- | --- | --- |
| HTML to PDF rendering, e.g. `/render/html` | No | OpenAPI has no HTML render endpoint. `POST /render/html` returned `405`. |
| Template upload for new `.docx` templates | Yes | `POST /templates` accepted `templates/bill.docx` and returned `201` with `templateKey`, `filePath`, and `previewFilePath`. |
| DOCX/PDF to PNG image conversion | No | OpenAPI has no conversion or PNG export endpoint. `POST /convert`, `/convert/docx/png`, and `/convert/pdf/png` returned `405`. |

## Ticket 019 Rendering Recommendation

Use Puppeteer for HTML to PDF in Ticket 019.

Qorstack is useful for stored DOCX/XLSX template rendering, but the public API does not expose HTML input rendering. For HTML-driven bills, Puppeteer is the shorter and more direct path: render the app-owned HTML, generate the PDF, and avoid translating HTML into Qorstack's DOCX template model.

## Ticket 017 Conversion Recommendation

Use a local CLI conversion path instead of Qorstack for image previews.

Qorstack generates PDF previews for uploaded templates and renders DOCX/XLSX templates to downloadable files, but it does not expose DOCX/PDF to PNG conversion. For Ticket 017, use a local conversion pipeline such as LibreOffice for DOCX-to-PDF plus a PDF-to-image CLI if PNG output is required.

```sh
curl "$QORSTACK_API_URL/docs"
curl "$QORSTACK_API_URL/swagger"
curl "$QORSTACK_API_URL/api-docs"
curl "$QORSTACK_API_URL/openapi/v1.json"
curl -H "X-API-Key: $QORSTACK_API_KEY" "$QORSTACK_API_URL/templates"
curl -H "X-API-Key: $QORSTACK_API_KEY" -F "file=@templates/bill.docx;type=application/vnd.openxmlformats-officedocument.wordprocessingml.document" "$QORSTACK_API_URL/templates?name=Ticket%20014%20Probe&templateKey=ticket-014-probe&isAutoGeneratedVariable=false"
curl -H "X-API-Key: $QORSTACK_API_KEY" -H "Content-Type: application/json" -d '{"templateKey":"ticket-014-probe","fileName":"ticket-014-probe.pdf","fileType":"pdf","replace":{"roomNumber":"101","billingMonth":"June 2026","tenantName":"Probe"}}' "$QORSTACK_API_URL/render/word/template"
curl -H "X-API-Key: $QORSTACK_API_KEY" -H "Content-Type: application/json" -d '{}' "$QORSTACK_API_URL/render/html"
curl -H "X-API-Key: $QORSTACK_API_KEY" -H "Content-Type: application/json" -d '{}' "$QORSTACK_API_URL/convert"
curl -H "X-API-Key: $QORSTACK_API_KEY" -H "Content-Type: application/json" -d '{}' "$QORSTACK_API_URL/convert/docx/png"
curl -H "X-API-Key: $QORSTACK_API_KEY" -H "Content-Type: application/json" -d '{}' "$QORSTACK_API_URL/convert/pdf/png"
```
